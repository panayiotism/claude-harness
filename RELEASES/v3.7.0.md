# Release v3.7.0 - TDD Enforcement Command

## What's Changed

### New Features
- **TDD Enforcement Command** (#23) @panayiotism
  - New `/claude-harness:do-tdd` command for test-driven development
  - Enforces RED-GREEN-REFACTOR workflow
  - Blocks implementation until test files exist
  - Keeps `/do` unchanged for backward compatibility

### New 7-Command Structure
| Command | Purpose |
|---------|---------|
| `/setup` | Initialize harness (one-time) |
| `/start` | Compile context, show status |
| `/do` | Unified workflow (features + fixes) |
| **`/do-tdd`** | **TDD workflow (tests first)** |
| `/checkpoint` | Manual commit + push + PR |
| `/orchestrate` | Multi-agent team (advanced) |
| `/merge` | Merge PRs, auto-version, release |

### Key Differences: `/do` vs `/do-tdd`
| Aspect | `/do` | `/do-tdd` |
|--------|-------|-----------|
| Test timing | After implementation | BEFORE implementation |
| Test check | Verification only | Existence gate + verification |
| Phase order | Plan â†’ Implement â†’ Verify | Plan â†’ Tests â†’ Implement â†’ Verify |
